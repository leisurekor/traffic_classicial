# 答辩问答建议

## 1. 这个项目到底是在做分类，还是在做异常检测？
建议回答：从对外任务形式看，它表现为二分类，即 benign / malicious；但从方法论核心看，它不是传统监督分类，而是 benign-only 的无监督异常检测。训练阶段只用 benign 建立正常参考分布，再根据偏离程度进行检测。

## 2. 为什么不直接做一个普通的监督分类器？
建议回答：真实恶意流量场景中，恶意标签通常稀缺、不完备且具有强开放性。纯监督分类依赖训练集覆盖未来恶意分布，这在现实中很难满足。因此，benign-only 异常检测更符合“正常样本相对可获得，恶意模式不断变化”的现实条件。

## 3. 为什么要同时做 CSV line 和 Graph line？
建议回答：两条线回答的是两个不同问题。CSV line 用来验证模型在标准流特征输入上的分类能力，尽量减少工程噪声；Graph line 用来验证从原始 PCAP 到检测的链路是否可行，以及图表示是否能提供结构价值。

## 4. 为什么图结构表示是必要的？
建议回答：恶意行为不一定在单个流的统计特征上特别显著，但往往会体现在端点交互结构、局部拓扑关系和时间上的联合模式中。图表示可以把“单样本异常”提升为“局部交互结构异常”。

## 5. 图主线的核心模型是什么？
建议回答：图主线的核心模型是 Graph AutoEncoder（GAE）。它由图编码器、节点解码器和边解码器组成，训练目标是重构 benign 图的节点与边特征，利用重构残差作为异常证据。

## 6. 为什么节点不是纯 IP？
建议回答：如果节点只用 IP 表示，会丢失端口、协议和角色信息。项目中采用 `endpoint_type + ip + port + protocol` 的方式定义节点，使其更接近实际网络服务端点，也更利于建模其结构角色。

## 7. communication edge 和普通图里的边有什么区别？
建议回答：它不是简单的“存在连通关系”，而是承载了丰富行为信息的对象，包括包数、字节数、时长、IAT 分布、TCP flag、burst proxy、ACK delay proxy 等，因此它更像“通信行为摘要”。

## 8. 图级分数为什么不能简单取平均？
建议回答：恶意行为在图中往往不是均匀分布的，而是集中体现在少量关键边或局部结构上。如果简单平均，局部强异常可能会被背景部分淹没。因此当前正式图主线更关注 high-anomaly tail 的局部证据。

## 9. merge01.csv 的高分说明了什么？
建议回答：它主要说明模型在标准化流特征输入上的分类能力已经被稳定验证。也就是说，在输入已经被整理为高质量流特征表时，现有方法确实能够有效地区分 benign 与 malicious。

## 10. CTU13 48 / 49 / 52 的结果到底意味着什么？
建议回答：它说明从原始 PCAP 出发，经由 flow、graph、feature 再到 detection 的链路是真实可行的，并且图主线在该 benchmark 上已经取得高质量结果。因此它可以被表述为 PCAP-based graph benchmark。

## 11. 为什么不能说“完整 PCAP 主实验体系已经完成”？
建议回答：因为当前正式结果只覆盖 CTU13 48 / 49 / 52，并没有形成跨数据集、系统化、可横向对比的完整 PCAP 实验矩阵。当前可以证明的是“已有 CTU13 的 PCAP-based graph benchmark 结果”，而不是“完整 PCAP 体系已完成”。

## 12. 当前项目最大的残余问题是什么？
建议回答：当前主要残余问题已经不是“完全抓不到 malicious”，而是 unknown / background 与 malicious 的重叠仍然较强。这会在图主线中表现为 background hit ratio 偏高。

## 13. nuisance-aware 为什么没有纳入最终正式主线？
建议回答：nuisance-aware 的研究动机是合理的，因为它直面复杂背景流量问题；但当前结果表明，它尚未形成比现有 single-stage 主线更稳定、可直接纳入正式主结果的方案，因此目前只能作为研究探索方向保留。

## 14. two-stage micrograph 为什么失败？
建议回答：失败的关键不在 verifier，而在 proposal / extraction 没能稳定提纯 malicious 局部结构。也就是说，下游判别器看到的 micrograph 与 unknown 仍然高度重叠，导致整体效果不稳定。

## 15. episode-graph 为什么没有继续成为主线？
建议回答：因为 episode proposal / sessionization 没能稳定构造出 malicious episode，整条路线容易退化成 reject-all 或近似 reject-all。因此它更适合被写成研究探索与失败分析，而不是正式贡献。

## 16. 当前结果足以支撑结项吗？
建议回答：可以，但必须严格控制结论边界。当前结果足以支撑“CSV 主结果成立”和“CTU13 图侧主 benchmark 成立”，也足以支撑“分类能力与图侧链路可行性”这两个核心结论；但不能夸大为完整 PCAP 主实验体系已经建立。
